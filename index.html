Basic Reactivity

The key idea of reactive programming is to specify a graph of dependencies so that when an input changes, all related outputs are automatically updated.

Wickham (2019) in Mastering Shiny
Warum ist das wichtig? Weil sonst das ganze R Script bei jeder Veränderung im App gesamthaft ausgeführt werden würde
aus Wickham (2019), Mastering Shiny

This makes the flow of an app considerably simpler, but it takes a while to get your head around how it all fits together.

Futher reading:

  • mastering-shiny.org/basic-reactivity.html
  • mastering-shiny.org/reactivity-intro.html

Reactive Outputs 

						library(shiny)

						ui <- plotOutput("plot")

						server <- function(input, output, session){
							output$plot <- renderPlot(plot(rnorm(100)))
						}


						shinyApp(ui, server)
Reactive Outputs kommen immer in paaren. render* um den output zu generieren, *Output um den output zu rendern.
